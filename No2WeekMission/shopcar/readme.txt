***********************************************下面涉及到的5个文件需要事先全部手动建立好****************************************
# 代码中需要手动改写文件路径
accountManager.txt文件：（默认内容为空，进入消费时根据提示手动开户存钱进去）
	主要存放用户账户信息，姓名和余额两个字段组成。多人账户共存于这个文件
	格式：姓名+“余额”+ 余额
	{"zhangtong": {"余额": 3787}, "alex": {"余额": 4347}, "king": {"余额": 704}}

buyRecords.txt文件：(默认内容为空，消费成功后自动记录)
	用于存放所有用户的消费记录信息，格式如下：
姓名1+
单号+消费时间+消费商品信息（数量,单价）,
单号+消费时间+消费商品信息（数量,单价）,
姓名2+
单号+消费时间+消费商品信息（数量,单价）,
单号+消费时间+消费商品信息（数量,单价）,
以JSON格式存放
{"zhangtong": {"1141880000": {"消费时间": "2017-07-12 22:58:59", "饼干": [1, 45]}, "1141880001": {"消费时间": "2017-07-12 22:59:12", "巧克力": [1, 80]}, "1141880003": {"消费时间": "2017-07-12 23:00:42", "笔记本电脑": [1, 299]}}, "alex": {"1141880002": {"消费时间": "2017-07-12 22:59:49", "足球": [1, 55], "笔记本电脑": [1, 299]}}}

consumerBarcode.txt文件：（默认内容为空）
	用于记录生成订单编号（每一单消费累加1保证每一单编号唯一）
	格式：
		程序之行一次后默认从"1141880000"开始累加

goods.txt文件：（此文件需要根据下列格式先写如商品）
	商品信息存放文件。
	JSON格式：编号+名称+单价+库存量
	[0, ["牛奶", 20, 92], 1, ["饼干", 45, 89], 2, ["巧克力", 80, 94], 3, ["足球", 55, 97], 4, ["电饭煲", 4999, 34], 5, ["笔记本电脑", 299, 897]]

users.txt文件：（此文件需要根据下列格式先录入用户信息）
	用户登陆信息存放文件
	JSON格式：姓名+余额+登陆失败次数
	{"zhangtong": ["666", 0], "alex": ["777", 0], "king": ["888", 0]}
